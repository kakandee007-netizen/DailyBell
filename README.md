# DailyBell 🔔

A simple, clean daily-routine reminder app for iPhone, built with SwiftUI.

DailyBell is **not just for skincare** — add anything you do each day: skincare,
studying, exercise, work, drinking water, sleeping, project tasks, personal
reminders, and more. Each item repeats every day at the time you choose.

- **Name:** DailyBell
- **Bundle ID:** `com.dailybell.app`
- **Display name:** DailyBell
- **Min iOS:** 16.0

---

## Features (v1)

- Add a reminder/task with **title**, **time**, optional **note**, and a **notification on/off** toggle.
- Today's reminders shown in a clean list **sorted by time**.
- **Local notifications** that repeat **every day** at the chosen time (`UNUserNotificationCenter`).
- Notification **permission requested on first launch**.
- **Mark as Done** (the done state resets automatically each day).
- **Delete** with swipe.
- **Local storage** with `Codable` + `UserDefaults` (nothing leaves the device).

## Build it yourself locally (optional)

You need a Mac with Xcode and [XcodeGen](https://github.com/yonaskolb/XcodeGen).

```bash
brew install xcodegen      # once
cd DailyBell
xcodegen generate          # creates DailyBell.xcodeproj
open DailyBell.xcodeproj    # build & run in Xcode
```

## Build an unsigned IPA with GitHub Actions

This produces an **unsigned** `.ipa` you can later sign with KSign on your iPhone.

1. **Create a GitHub repo** and push these files so the layout matches the tree
   above (the workflow must live at `.github/workflows/build-ipa.yml`).

   ```bash
   cd DailyBell
   git init
   git add .
   git commit -m "DailyBell v1"
   git branch -M main
   git remote add origin https://github.com/<your-username>/DailyBell.git
   git push -u origin main
   ```

2. **The build runs automatically** on every push to `main`. You can also trigger
   it manually: open the repo → **Actions** tab → **Build Unsigned IPA** →
   **Run workflow**.

3. **Download the IPA:** when the run finishes (green check), open the run and
   download the artifact named **`DailyBell-unsigned-ipa`**. Unzip it to get
   `DailyBell.ipa`.

### How the unsigned IPA is made

The workflow installs XcodeGen, generates the project, builds for a real device
with code signing disabled (`CODE_SIGNING_ALLOWED=NO`), then packages the `.app`
into a standard `Payload/` folder and zips it as `DailyBell.ipa`.

## Sign & install with KSign

KSign signs the IPA on-device using your own certificate/provisioning profile,
then installs it.

1. Transfer `DailyBell.ipa` to your iPhone (AirDrop, Files app, iCloud Drive,
   or download it from GitHub directly on the phone).
2. Open **KSign**.
3. Import your signing assets in KSign:
   - your **certificate** (`.p12`) and its password, **or** an Apple Developer
     account, depending on how your KSign is set up, plus
   - a matching **provisioning profile** (`.mobileprovision`) if required.
4. Choose **DailyBell.ipa** as the app to sign.
5. Tap **Sign**, then **Install**. Approve the install prompt.
6. On first launch, allow **Notifications** so reminders can fire.

> If iOS shows "Untrusted Developer," go to
> **Settings → General → VPN & Device Management** and trust your developer
> profile.

## Notes & tips

- **Daily behavior:** every reminder repeats daily. The "Done" checkmark is for
  today and clears at midnight, so each day starts fresh.
- **Background notifications** are scheduled by the system and fire even if the
  app is closed, as long as notification permission is granted.
- **Bundle ID / signing:** the bundle ID is `com.dailybell.app`. If your signing
  profile requires a different ID, change `PRODUCT_BUNDLE_IDENTIFIER` in
  `project.yml`, re-run `xcodegen generate`, and rebuild.
- **Xcode version:** the workflow uses the default Xcode on the `macos-14`
  runner. If a build ever fails due to version drift, pin a version by adding a
  step like `sudo xcode-select -s /Applications/Xcode_15.4.app` before the build,
  or change `runs-on` to `macos-15`.
